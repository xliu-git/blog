MCP小结

什么是MCP MCP是一个标准化协议（protocol），规定了应用和大模型之间的通信格式，比如我们常说的USB-C就是一种协议，同一种协议，不同厂家的具体实现可能有很多种，比如USB2.0 Type-C, USB3.1 Gen1 Type-C, Thunderbolt3/4 Type-C等。 MCP宏观架构 我根据MCP introduction的图，以及官方给出的MCP Server和Client的例子，结合自己的理解画了一张图。 Host:一个应用，或者更抽象，应用中的一个connection pool，维护着各种MCP Client的connection instances。 MCP Client:我在这里把client理解成一个跟其他通信协议类似的client，比如http client、webSocket client，负责跟具体server的connection。 MCP Server:即提供相应服务的“server”，这里的server可能只是一层调用外部service provider API的封装，当然，内部的依赖也可以有native的MCP Server。 Orchestration Service:负责组合各种MCP Client的tool，决定什么时候调用哪个MCP client的哪种tool，并通过返回的response进行下一步的工作。 MCP Server例子 我最开始对于MCP Server的理解是Service Provider专门针对MCP协议所部属的server，那么所有的MCP Server的开发工作应该属于Service Provider，但查看了一下github为Claude给出的MCP Server例子后发现MCP Server其实更类似于一种本地对于service provider的封装，或者说专门针对MCP协议路由的controller，提供MCP协议要求的服务，至于服务的具体实现，则是通过service provider原有的API call实现的。 以github的MCP server为例： 提供tool列表、描述以及schema：https://github.com/modelcontextprotocol/servers/blob/main/src/github/index.ts#L71 提供tool路由：https://github.com/modelcontextprotocol/servers/blob/main/src/github/index.ts#L208 运行tool（通过API实现）：https://github.com/modelcontextprotocol/servers/blob/main/src/github/operations/commits.ts#L12 Reference https://modelcontextprotocol.io/introduction
